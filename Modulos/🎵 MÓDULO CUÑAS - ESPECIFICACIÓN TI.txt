📋 DESCRIPCIÓN Y PROPÓSITO DEL MÓDULO
¿QUÉ ES EL MÓDULO CUÑAS?
El Módulo Cuñas es el centro neurálgico de gestión de contenido publicitario en Silexar Pulse. Funciona como el repositorio inteligente donde convergen todos los activos creativos de audio, textos de locución, presentaciones, cierres y contenido promocional que alimentan las campañas publicitarias.
PERSPECTIVA DEL USUARIO DIARIO:
Como Programador/Traffic Manager, paso el 60% de mi día en este módulo:

Recibo cuñas de clientes por email/WhatsApp y las subo al sistema
Valido que las presentaciones coincidan con los auspicios en Vencimientos
Programo horarios de emisión y verifico cumplimiento
Genero menciones cuando no hay audio pregrabado
Envío paquetes de cuñas a operadores de emisión

Como Ejecutivo de Ventas, uso este módulo para:

Mostrar al cliente las cuñas disponibles durante reuniones comerciales
Crear presentaciones personalizadas para nuevos auspicios
Validar que tenemos todo el material antes de confirmar contratos
Enviar previews de cuñas a clientes para aprobación

Como Gerente Comercial, superviso:

Qué clientes no han entregado material creativo
Cuáles cuñas están por vencer y necesitan renovación
La calidad y consistencia del material publicitario
El cumplimiento de entregas vs cronogramas de campaña

INTEGRACIÓN CON ECOSYSTEM SILEXAR:

Módulo Contratos: Al crear un contrato, automáticamente se valida si existen cuñas activas para el anunciante
Módulo Vencimientos: Las presentaciones y cierres se validan contra auspicios programados
Módulo Emisoras: Las cuñas se exportan automáticamente a sistemas de aire (WideOrbit, Sara, Dalet)
Módulo Facturación: Solo se factura lo que efectivamente tiene material para emitir
Cortex-Sense: Verifica automáticamente que las cuñas se emitieron según lo programado

typescriptconst processAudioFileAdvanced = async (file: File, cunaId: string) => {
  const processingId = generateProcessingId();
  
  // UI: Mostrar progreso inmediatamente
  showProcessingProgress(processingId, {
    fileName: file.name,
    fileSize: file.size,
    status: 'uploading'
  });
  
  try {
    // 1. Upload con progress tracking
    const uploadUrl = await uploadToGCS(file, {
      onProgress: (percent) => updateProgress(processingId, 'uploading', percent)
    });
    
    updateProgress(processingId, 'analyzing', 0);
    
    // 2. Análisis técnico completo
    const technicalAnalysis = await analyzeAudioTechnical(uploadUrl);
    const contentAnalysis = await analyzeAudioContent(uploadUrl);
    
    updateProgress(processingId, 'analyzing', 50);
    
    // 3. Validaciones automáticas
    const qualityCheck = await validateAudioQuality(technicalAnalysis);
    const brandSafetyCheck = await validateBrandSafety(contentAnalysis);
    
    updateProgress(processingId, 'analyzing', 100);
    updateProgress(processingId, 'finalizing', 0);
    
    // 4. Generación de metadatos
    const metadata = {
      ...technicalAnalysis,
      qualityScore: qualityCheck.score,
      brandSafety: brandSafetyCheck.rating,
      fingerprint: await generateAudioFingerprint(uploadUrl),
      waveform: await generateWaveform(uploadUrl)
    };
    
    // 5. Auto-actualización del formulario
    await updateCunaFields({
      id: cunaId,
      duration: Math.ceil(technicalAnalysis.duration),
      audioUrl: uploadUrl,
      metadata: metadata,
      status: qualityCheck.approved ? 'ready' : 'needs_review'
    });
    
    updateProgress(processingId, 'complete', 100);
    
    // 6. Abrir editor automáticamente si se requiere
    if (qualityCheck.needsEditing) {
      openAudioEditor(uploadUrl, metadata);
    }
    
    hideProcessingProgress(processingId);
    showSuccessNotification("Audio procesado exitosamente", {

typescriptconst analyzeTextForSpeech = async (text: string) => {
  // Limpieza y normalización
  const cleanText = text.replace(/\[.*?\]/g, ''); // Remover marcadores
  const words = cleanText.split(/\s+/).filter(Boolean);
  const syllables = await countSyllables(cleanText);
  
  // Análisis de complejidad
  const complexityFactors = {
    numbers: (text.match(/\d+/g) || []).length,
    brands: await detectBrandNames(text),
    foreignWords: await detectForeignWords(text),
    acronyms: (text.match(/[A-Z]{2,}/g) || []).length,
    punctuation: (text.match(/[.!?;,:]/g) || []).length
  };
  
  // Cálculo de velocidad base
  let baseWPM = 150; // Velocidad estándar radio comercial
  
  // Ajustes por complejidad
  if (complexityFactors.numbers > 3) baseWPM -= 15;
  if (complexityFactors.brands.length > 2) baseWPM -= 10;
  if (complexityFactors.foreignWords.length > 0) baseWPM -= 20;
  if (complexityFactors.acronyms > 1) baseWPM -= 10;
  
  // Análisis de elementos especiales
  const specialElements = {
    emphasis: (text.match(/\[ÉNFASIS\].*?\[\/ÉNFASIS\]/g) || []).length,
    pauses: (text.match(/\[PAUSA:(\d+(?:\.\d+)?)s?\]/g) || []).length,
    spelling: (text.match(/\[DELETREO\].*?\[\/DELETREO\]/g) || []).length
  };
  
  // Cálculo de tiempo adicional por elementos especiales
  let additionalTime = 0;
  additionalTime += specialElements.emphasis * 0.5; // 0.5s por énfasis
  additionalTime += specialElements.spelling * 3; // 3s por deletreo
  
  // Tiempo base de lectura
  const baseReadingTime = (words.length / baseWPM) * 60;
  
  // Tiempo total estimado
  const totalTime = baseReadingTime + additionalTime;
  
  // Rango de variación (±15% para variabilidad humana)
  const minTime = Math.ceil(totalTime * 0.85);
  const maxTime = Math.ceil(totalTime * 1.15);
  
  return {
    wordCount: words.length,
    characterCount: text.length,
    syllableCount: syllables,
    baseWPM,
    adjustedWPM: Math.round((words.length / baseReadingTime) * 60),
    estimatedSeconds: Math.ceil(totalTime),
    timeRange: `${minTime}-${maxTime}s`,
    complexityLevel: calculateComplexityLevel(complexityFactors),
    specialElements,
    suggestions: generateLocutionSuggestions(text, complexityFactors)
  };
};

      alerts.push({
        type: 'critical_expirations',
        count: criticalExpirations.length,
        items: criticalExpirations,
        suggestedActions: generateExpirationActions(criticalExpirations),
        priority: 'critical'
      });
    }
    
    // Análisis de calidad técnica
    const qualityIssues = await analyzeQualityTrends();
    if (qualityIssues.failureRate > 0.05) {
      alerts.push({
        type: 'quality_degradation',
        failureRate: qualityIssues.failureRate,
        trendDirection: qualityIssues.trend,
        mainCauses: qualityIssues.primaryCauses,
        priority: 'high'
      });
    }
    
    // Análisis de performance del equipo
    const teamPerformance = await analyzeTeamPerformance();
    if (teamPerformance.hasImbalances) {
      alerts.push({
        type: 'workload_imbalance',
        imbalanceLevel: teamPerformance.variancePercentage,
        overloadedUsers: teamPerformance.overloaded,
        underutilizedUsers: teamPerformance.underutilized,
        priority: 'medium'
      });
    }
    
    return alerts;
  }
